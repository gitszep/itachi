🧙‍♂️ Święty Graal Gita – Pełny Przebieg Pracy

🔗 Połączenie ze zdalnym repozytorium

git remote add origin url

🌱 Tworzenie nowej gałęzi roboczej

git checkout -b galaz

🛠️ Wprowadź zmiany i je zatwierdź

git add .
git commit -m "Opis zmian"

⬅️ Powrót na główną gałąź i scalenie

git checkout main
git merge galaz

💾 Dodaj i zatwierdź dodatkowe zmiany (jeśli są)

git add .
git commit -m "Opis zmian"

⬇️ Pobierz najnowsze zmiany zdalne

git pull origin main

⬆️ Wypchnij zmiany do zdalnego repozytorium

git push origin main

🚨 Jeśli merge nie działa – RĘCZNE ROZWIĄZANIE

# Wejdź na itachi
git checkout itachi

# Skopiuj plik
cp index.html /tmp/index.html

# Przejdź na main
git checkout main

# Nadpisz plik
cp /tmp/index.html index.html

# Zapisz zmianę
git add index.html
git commit -m "Wrzucenie kodu HTML z itachi na main"

📜 Jak sprawdzić historię commitów

git log --oneline

📖 Skrócona wersja – Święty Graal Gita

git remote add origin url
git checkout -b galaz
git add .
git commit -m "Opis zmian"
git checkout main
git merge galaz
git add .
git commit -m "Opis zmian"
git pull origin main
git push origin main

🔁 Podstawowe komendy Git

git checkout nazwa-gałęzi         # Przełącz gałąź
git push origin nazwa-gałęzi      # Wypchnij zmiany
git branch -a                     # Lista lokalnych i zdalnych gałęzi
git checkout -b nowa-galaz        # Nowa gałąź
git branch -D nazwa-gałęzi        # Usuń lokalną gałąź
git fetch --prune                 # Odśwież gałęzie zdalne
git push origin --delete Main     # Usuń gałąź zdalną
